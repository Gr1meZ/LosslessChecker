# LosslessChecker — полное руководство по анализу и метрикам

## Что делает программа

LosslessChecker анализирует аудиофайлы и отвечает на три вопроса:
1. **Аутентичность** — настоящий ли это lossless, или пережатый из lossy (MP3/AAC) в lossless-контейнер (FLAC/WAV)?
2. **Качество мастеринга** — насколько хорошо смастерен трек (клиппинг, громкость, фаза)?
3. **Решение** — ОСТАВИТЬ, ПРОВЕРИТЬ, или ЗАМЕНИТЬ?

## Поддерживаемые форматы

| Формат | Тип | Что проверяем |
|--------|-----|---------------|
| FLAC | Lossless | Подлинность, транскод из lossy, битрейт, сжатие |
| WAV | Lossless | Подлинность, транскод, fake 24-bit |
| ALAC (M4A) | Lossless | Подлинность, транскод |
| MP3 | Lossy | Качество рипа (честность битрейта, артефакты) |
| AAC (M4A) | Lossy | Качество рипа, артефакты |

Не поддерживаются: OGG, Opus, WavPack, TTA, APE, DTS-CD.

## Как работает анализ (по шагам)

### Шаг 1: Чтение формата
Программа читает заголовок файла, извлекает:
- **Sample Rate** (частота дискретизации): 44100 Гц (CD), 48000 Гц, 96000 Гц и т.д.
- **Bit Depth** (битовая глубина): 16 бит, 24 бит
- **Channels** (каналы): моно (1) или стерео (2)
- **Битрейт** (для MP3/AAC): из заголовка Xing для MP3, из MP4-атомов для AAC

### Шаг 2: Декодирование
Аудио читается через NAudio в сырые float-сэмплы. Стерео разделяется на левый и правый каналы.

### Шаг 3: Спектральный анализ
FFT (4096 точек) по всему треку с шагом 2048 сэмплов. Определяются:

- **Cutoff Frequency** — частота, выше которой сигнал отсутствует или резко падает
- **Shelf Type** — форма спада (кирпичная стена / фильтрованный / естественный)
- **Artifacts** — следы lossy-сжатия (плоский шум выше среза, спектральные дыры)
- **Pre-echo** — шум перед резкими атаками (характерно для MP3/AAC)

### Шаг 4: Динамический анализ
- **DR (Dynamic Range)** — разница между пиковым и средним уровнем в 20% самых громких 3-секундных блоков
- **True Peak** — межсэмпловый пик через 4× оверсэмплинг (ITU-R BS.1770-4)
- **Sample Peak** — цифровой пик (максимальное значение сэмпла)
- **Clipping** — процент сэмплов, достигших цифрового потолка (3+ подряд = клип)
- **LUFS** — интегрированная громкость по стандарту вещания (ITU-R BS.1770-4)
- **PLR** — Peak-to-Loudness Ratio (True Peak минус LUFS)

### Шаг 5: Технические проверки
- **Bit Depth** — соответствие заявленной битности реальной (LSB zero-padding)
- **DC Offset** — постоянная составляющая сигнала
- **Фаза** — корреляция левого/правого каналов, моно-совместимость
- **Upscale** — апскейл из CD в Hi-Res (отсутствие контента выше 22 кГц)
- **Vinyl** — следы винила (низкочастотный рокот, ВЧ-шум)
- **MQA/HDCD** — специальные форматы кодирования

### Шаг 6: Расчёт битрейта и сжатия
- **Средний битрейт** = (размер файла − метаданные) × 8 / длительность
- **Степень сжатия** = размер файла / несжатый размер (для FLAC/WAV)
- **Фреймовый анализ** (только FLAC) — мгновенный битрейт каждого фрейма
- **Подозрительный битрейт** — флаг если битрейт не соответствует формату

### Шаг 7: Скоринг
Три независимые оценки:
1. **Lossless Score** (0–100) — подлинность lossless
2. **Quality Score** (0–100) — качество мастеринга
3. **Hi-Res Score** (0–100) — подлинность Hi-Res (только для ≥88.2 кГц)

## Метрики: что значит каждая

### Спектральный анализ

#### Частотный срез (Cutoff)
Максимальная частота, выше которой сигнал отсутствует. Настоящий lossless сохраняет полный спектр до частоты Найквиста (половина Sample Rate).

| Cutoff (% Найквиста) | Что значит |
|---------------------|------------|
| >95% | Отлично — полный спектр, lossless |
| 85–95% | Подозрительно — возможен MP3 320 или AAC 256 |
| <85% | Точно сжато — MP3 128-256 |

**Важно для 48 кГц**: MP3 при 48 кГц режет максимум до 20.5 кГц (85% Найквиста). Кирпичная стена выше — не кодек, а мастеринговый LP-фильтр.

#### Характер спада (Shelf Type)
Форма спектра выше частоты среза:

| Тип | Что значит | Хорошо/Плохо |
|-----|-----------|-------------|
| Естественный | Плавное затухание — аналоговая запись или качественный цифровой трансфер | ✓ Хорошо |
| Фильтрованный | Мягкий спад — мастеринговый LP-фильтр | ⚠ Нейтрально |
| Кирпичная стена | Резкий обрыв — признак lossy-кодека | ✗ Плохо (на 44.1 кГц), ⚠ на 48 кГц при >90% Найквиста |

#### Артефакты сжатия
Характерные искажения lossy-кодеков: спектральная полка шума, блоковые артефакты, аномальная плоскостность выше среза.

| Уровень | Что значит |
|---------|-----------|
| None | Чисто — нет следов lossy |
| Weak | Слабые — возможно, был сжат, но не точно |
| Medium | Средние — вероятно, пережат из lossy |
| Strong | Сильные — точно пережат из lossy |

### Динамика и громкость

#### Динамический диапазон (DR)
Разница между пиковым и средним уровнем в самых громких участках. Измеряется по методу TT DR Meter (20% самых громких 3-секундных блоков).

| DR | Что значит | Типичные жанры |
|----|-----------|---------------|
| 12+ | Аудиофил | Джаз, классика, акустика, винил |
| 8–11 | Золотая середина | Рок 80-90х, инди, симфо-метал |
| 5–7 | Плотный звук | Современный метал, альт-рок, поп |
| 3–4 | Кирпичная стена | EDM, экстрим-метал, гиперпоп |

**Важно**: низкий DR не всегда плохо — для EDM или современного метала DR3-5 это норма. DR сам по себе не делает файл фейком.

#### True Peak (межсэмпловый пик)
Реальный пик аналогового сигнала между цифровыми сэмплами. Измеряется через 4× оверсэмплинг с 64-tap FIR-фильтром (стандарт ITU-R BS.1770-4).

| True Peak | Что значит |
|-----------|-----------|
| <0 dBTP | Чисто — нет межсэмпловых перегрузок |
| 0..+1 dBTP | Искажения — возможны артефакты при конвертации |
| >+1 dBTP | Сильный перегруз — гарантированные искажения |

**Почему True Peak ≠ Sample Peak**: два соседних сэмпла могут быть ниже 0 dBFS, но аналоговый сигнал между ними — выше. True Peak ловит эти невидимые перегрузки.

#### Интегрированная громкость (LUFS)
Средняя воспринимаемая громкость по стандарту ITU-R BS.1770-4. Учитывает частотную чувствительность слуха (K-weighting: +4 dB от 1682 Гц, спад ниже 38 Гц).

| LUFS | Что значит |
|------|-----------|
| <−16 | Динамично — аудиофильский мастеринг |
| −14 | Стриминг-цель (YouTube, Spotify нормализуют к −14) |
| −11..−8 | Коммерческий стандарт (CD-мастеринг) |
| >−7 | Экстремально громко — жертва войны громкости |

#### PLR (Peak-to-Loudness Ratio)
Разница между True Peak и Integrated LUFS. Показывает сохранность макродинамики.

| PLR | Что значит |
|-----|-----------|
| >8 дБ | Отлично — есть динамические перепады |
| 6–8 дБ | Сжато — макродинамика пострадала |
| <6 дБ | Пережато — трек лишён динамических перепадов |

### Технические параметры

#### Битовая глубина
Соответствие заявленной битности реальной.

| Ситуация | Что значит |
|----------|-----------|
| 16 бит, эфф. 16 бит | Честно — все 16 бит используются |
| 24 бит, эфф. 24 бит | Честно — настоящий 24-битный файл |
| 24 бит, эфф. 16 бит | **Фейк** — младшие 8 бит всегда нули. Это 16-битный файл, сохранённый в 24-битном контейнере. Размер вдвое больше без прироста качества |

#### DC смещение
Постоянная составляющая сигнала. Должна быть близка к 0.

| DC Offset | Что значит |
|-----------|-----------|
| <0.01% | Норма |
| >0.01% | Дефект оцифровки — съедает динамический диапазон, вызывает щелчки |

#### Фаза / Стереокорреляция
Степень синфазности левого и правого каналов.

| Корреляция | Что значит |
|-----------|-----------|
| 0..+1.0 | Норма — от моно до широкого стерео |
| <0 | Проблема — противофаза. Звук пропадёт при моно-воспроизведении |

#### Fake Stereo
Оба канала идентичны (корреляция >0.99, задержка <1 сэмпл). Фактически моно, сохранённое как стерео — занимает вдвое больше места.

#### Обрезанные края (Abrupt Edges)
Первые/последние 500 мс трека имеют RMS ниже −60 dBFS при общем RMS выше −30. Вероятно, трек обрезан не по нулевой точке — может быть слышен щелчок.

#### ReplayGain расхождение
Тег REPLAYGAIN_TRACK_GAIN (если есть) расходится с измеренным Integrated LUFS более чем на 3 dB.

### Битрейт и сжатие

#### Средний битрейт
Реальный средний битрейт: (размер файла − метаданные) × 8 / длительность. Вычитаются ID3v2-теги (MP3) и FLAC-метаданные.

#### Степень сжатия
Размер файла / несжатый размер. Только для FLAC/WAV.

| Сжатие | Что значит |
|--------|-----------|
| 50–70% | Норма — типичное FLAC-сжатие |
| 70–90% | Слабое сжатие — сложный сигнал или плохое сжатие |
| >95% | Подозрительно — FLAC почти не сжимает. Возможно, внутри уже lossy |

#### Битрейт по фреймам (только FLAC)
Мгновенный битрейт FLAC-фреймов (VBR). Показывает разброс между тихими и громкими участками. Типичный разброс — в 2–5 раз.

#### Подозрительный битрейт
Флаг срабатывает когда:
- Средний битрейт <600 kbps и при этом кирпичная стена на <21 кГц (транскод)
- 24-битный файл имеет эффективную битность ≤16
- Степень сжатия >95% для 16-битного файла

## Как работает скоринг

### Lossless Score (подлинность)

Стартует со 100 баллов. Вычитаются штрафы:

| Штраф | Баллы | Условие |
|-------|-------|---------|
| Кирпичная стена <65% Найквиста | −60 | Явный MP3 128 |
| Кирпичная стена <75% Найквиста | −55 | MP3 128-192 |
| Кирпичная стена <85% Найквиста | −35 | MP3 192-256 |
| Кирпичная стена <90% Найквиста | −20 | MP3 256-320 |
| Кирпичная стена <95% Найквиста | −8 | Возможный MP3 320 |
| Strong артефакты + Brickwall | −35 | Точно lossy |
| Medium артефакты + Brickwall | −20 | Вероятно lossy |
| Weak артефакты | −8 | Подозрительно |
| LSB zero-pad + 24-bit | −100 | Гарантированный FAKE 24-bit |
| LSB zero-pad + 16-bit | −20 | Подозрительно |
| Upscale | −25 | Апскейл в Hi-Res |

Добавляются бонусы:

| Бонус | Баллы | Условие |
|-------|-------|---------|
| FLAC integrity OK | +5 | MD5 чек-сумма совпадает |
| CD-aligned + FLAC OK | +5 | Выравнивание по CD-секторам (588 сэмплов) |

**Гарантия чистоты**: если срез ≥90% Найквиста, shelf не кирпичная стена, и артефактов нет — минимальный балл = 75. Это защищает чистые файлы с мастеринговыми LP-фильтрами от ложных флагов.

**Классификация по баллам:**

| Баллы | Authenticity |
|-------|-------------|
| ≥70 | **TRUE** — подлинный lossless |
| 50–70 | **UNCERTAIN** — подозрительный, проверьте источник |
| <50 | **FALSE** — не подлинный, ищите оригинал |

### Quality Score (качество мастеринга)

Стартует со 100 баллов. Штрафы за проблемы мастеринга:

| Штраф | Баллы | Условие |
|-------|-------|---------|
| Клиппинг >5% | −20 | Сильные искажения |
| Клиппинг >2% | −12 | Заметный клиппинг |
| Клиппинг >0.5% | −6 | Единичный клиппинг |
| Клиппинг >0% | −2 | Следы |
| ISP (межсэмпловый перегруз) | −8 | True Peak > 0 dBTP |
| ISP + True Peak > +1 dBTP | −5 дополнительно | Сильный межсэмпловый перегруз |
| LUFS > −5 | −15 | Экстремально громко |
| LUFS > −8 | −8 | Очень громко |
| LUFS > −14 | −3 | Громко |
| DC Offset > 1% | −8 | Сильное смещение |
| DC Offset > 0.5% | −3 | Заметное смещение |
| Фаза < −0.5 | −12 | Сильная противофаза |
| Фаза < 0 | −6 | Противофаза |
| LSB zero-pad | −5 | Fake битность |
| Fake Stereo | −5 | Моно в стерео |
| Обрезанные края | −3 | Щелчки на границах |

**Классификация:**

| Баллы | Decision |
|-------|----------|
| ≥80 | KEEP (Excellent) |
| ≥50 | KEEP (Good) |
| <50 | KEEP (Fair) |

**НО**: это только если Authenticity = TRUE. Если файл UNCERTAIN → INVESTIGATE. Если FALSE → REPLACE.

### Hi-Res Score (подлинность Hi-Res)

Только для файлов с частотой ≥88.2 кГц. Стартует со 100 баллов.

| Штраф | Баллы | Условие |
|-------|-------|---------|
| Max HF < −60 dB | −60 | Контента выше 22 кГц нет (апскейл из CD) |
| Max HF < −40 dB | −30 | Очень мало ВЧ |
| Max HF < −25 dB | −10 | Мало ВЧ |
| Cutoff ≤ 17 кГц | −40 | MP3 128 → Hi-Res (апскейл) |
| Cutoff ≤ 20 кГц | −20 | MP3 320 / AAC → Hi-Res |
| Cutoff ≤ 22.1 кГц | −40 | CD → Hi-Res (апскейл) |
| Upscale | −40 | Явный апскейл |

**Классификация:**

| Баллы | Что значит |
|-------|-----------|
| ≥70 | Настоящий Hi-Res |
| 40–70 | Сомнительно |
| <40 | Апскейл |

## Специальные случаи

### MP3/AAC — оценка качества рипа

Для lossy-файлов вместо Lossless Score используется специальная шкала качества рипа:

| Срез | Битрейт | Оценка |
|------|---------|--------|
| ≥20 кГц | 320 kbps / V0 | 100 |
| ≥19 кГц | 256 kbps / V2 | 90 |
| ≥18.5 кГц | 192 kbps | 80 |
| ≥16 кГц | 128 kbps | 50 |
| <16 кГц | <128 kbps | 20 |

**Транскод-детект**: если срез на 30% ниже ожидаемого для заявленного битрейта → оценка 0 (явный транскод).

**Штрафы**: сильные артефакты (−25), спектральные дыры (−15), расхождение заявленного и реального битрейта (−15–30).

### MQA
Если обнаружен MQA-контейнер → Authenticity = "MQA", Decision = "MQA (needs decoder)". Стандартный скоринг не применяется — MQA требует специального декодера.

### HDCD
Если обнаружено HDCD-кодирование (только для .flac и .wav, так как HDCD — технология физических CD):
- Информация добавляется в отчёт
- На скоринг не влияет
- M4A/MP3 файлы не проверяются на HDCD

### Транскод (MP3/AAC → FLAC/WAV)
Если в lossless-контейнере обнаружен lossy-спектр:
- Authenticity = "TRANSCODE (MP3→FLAC)" или "TRANSCODE (AAC→FLAC)"
- Decision = "REPLACE" (всегда)
- Quality = 0

### Транскод (формат .mp3/.m4a/.aac)
Если файл в родном lossy-контейнере (.mp3, .m4a, .aac) — это не транскод, а честный lossy. Оценивается по шкале качества рипа:
- ≥80 баллов → KEEP (хороший рип)
- ≥50 баллов → INVESTIGATE (средний рип)
- <50 баллов → REPLACE (плохой рип)

## Как интерпретировать результаты

### Пример 1: Чистый FLAC
```
Cutoff: 22000 Гц (99% Найквиста)
Shelf: Естественный спад
Artifacts: None
DR: 8
Quality: 94%
Authenticity: TRUE
Decision: KEEP (Excellent)
```
**Вывод**: отличный lossless-файл, хороший мастеринг, оставляем.

### Пример 2: Транскод из MP3 в FLAC
```
Cutoff: 16000 Гц (72% Найквиста)
Shelf: Кирпичная стена
Artifacts: Strong (MP3)
DR: 5
Quality: -
Authenticity: TRANSCODE (MP3→FLAC)
Decision: REPLACE
```
**Вывод**: это MP3, перепакованный в FLAC. Качество не улучшилось, только размер вырос. Ищем оригинал в настоящем lossless.

### Пример 3: Честный MP3 320
```
Cutoff: 20000 Гц (90% Найквиста)
Shelf: Кирпичная стена
Artifacts: Weak
DR: 6
Mp3 Quality: 85%
Authenticity: LOSSY (MP3)
Decision: KEEP
```
**Вывод**: честный MP3 320 kbps, хороший рип, можно оставить.

### Пример 4: Hi-Res с апскейлом
```
Cutoff: 22050 Гц
Shelf: Кирпичная стена
Max HF: -65 dB
HiRes Score: 20%
Authenticity: FALSE
Decision: REPLACE
```
**Вывод**: файл заявлен как 96 кГц, но контента выше 22 кГц нет — это CD, переконвертированный в Hi-Res. Качества не прибавилось.

## Технические детали (для интересующихся)

### Алгоритм поиска cutoff
Используется производный метод (derivative-based): вместо фиксированного порога −60 dB ищется **самый крутой отрицательный перепад** в dB-спектре. Скользящее окно 30 бинов, линейная регрессия в log-шкале.

- Slope < −18 dB/oct → кирпичная стена (lossy-кодек)
- Slope −10..−18 dB/oct → фильтрованный спад (LP-фильтр)
- Slope > −10 dB/oct → естественный спад

### Алгоритм True Peak
64-tap Kaiser-windowed sinc (β=7.5) для 4× оверсэмплинга. Полифазная декомпозиция (4 фазы × 16 коэффициентов). Каждая полифазная ветка независимо нормализована до DC gain = 1.0.

### Алгоритм LUFS (ITU-R BS.1770-4)
1. K-weighting: High-shelf +4 dB от 1682 Гц + High-pass от 38 Гц
2. Блоки 400 мс с 75% перекрытием (100 мс hop)
3. Среднеквадратичная мощность в каждом блоке → мгновенная громкость
4. Абсолютный гейт: блоки тише −70 LKFS отбрасываются
5. Относительный гейт: блоки тише средней −10 LU отбрасываются
6. Power-усреднение оставшихся блоков → Integrated LUFS

### Алгоритм DR (TT DR Meter)
1. Трек разбивается на 3-секундные блоки
2. Для каждого блока: RMS (среднеквадратичный уровень) и Peak (пиковая амплитуда)
3. Берутся 20% самых громких блоков (по RMS)
4. DR = средний Peak − средний RMS − 2.65 дБ (калибровочная константа)

### 48 кГц guard
На 48 кГц MP3 кодируется с cutoff не выше 20.5 кГц (85% Найквиста). Кирпичная стена выше этого — мастеринговый LP-фильтр, а не кодек. Программа автоматически переклассифицирует Brickwall → Filtered для 48 кГц при cutoff >90% Найквиста.
